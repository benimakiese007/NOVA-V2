# Documentation Skill

When creating or modifying modules, follow these documentation standards:

## Documentation Checklist

1. **JSDoc Headers**: Every function should have a JSDoc comment describing its purpose, parameters (@param), and return value (@returns).
2. **Inline Comments**: Use comments to explain "why" complex logic is implemented, not just "what" the code does.
3. **Module Overview**: Each new file should have a brief header comment explaining its responsibility.
4. **Consistency**: Ensure terminology matches existing documentation.

## Guidelines

- Keep documentation up to date with code changes.
- Avoid obvious comments (e.g., `i++ // increment i`).
- Use clear, professional language.
