# Refactoring Skill

Focus on making the codebase cleaner and more maintainable.

## Refactoring Checklist

1. **DRY (Don't Repeat Yourself)**: Extract duplicate logic into reusable functions or modules.
2. **Separation of Concerns**: Ensure components or functions have a single responsibility.
3. **Readability**: Simplify complex conditional logic and use descriptive variable names.
4. **Modern Standards**: Use modern JavaScript (ES6+) and CSS best practices.
5. **Dead Code**: Identify and remove unused variables, functions, or imports.

## Approach

- Propose refactors that reduce technical debt.
- Ensure that the refactor doesn't introduce regressions (verify with existing tests).
- Explain the benefits (e.g., "This makes the component easier to test").
