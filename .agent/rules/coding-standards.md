# Coding Standards

This rule ensures the project maintains its high-quality, framework-less, and premium aesthetic standards.

## Core Principles
1. **Pure Vanilla**: Use only Vanilla JavaScript and Plain CSS. No frameworks (React, Vue, etc.) or CSS libraries (Tailwind, Bootstrap) unless explicitly requested.
2. **Premium Design**: Every UI change must feel professional and high-end:
   - Use curated color palettes (no basic #f00, #00f).
   - Use modern typography (Google Fonts prefered).
   - Implement smooth transitions and micro-animations.
3. **Responsive**: All layouts must be mobile-first and fully responsive.

## JavaScript Best Practices
- Use ES6+ features (const/let, arrow functions, template literals).
- Avoid global variables; use modules or clean scoping.
- Ensure cross-browser compatibility.
